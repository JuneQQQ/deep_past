SimQAAgent 多智能体学术 QA 合成框架

用于生成学术写作 SFT 训练数据 · 基于 ReAct 模式 · 带质量过滤闭环

📄 输入：学术论文 PDF MinerU OCR 提取 → 段落切分 → 选取 excerpt
🎲 Task Selector 按预设比例随机分配任务类型（正向 / 逆向）

✅ 正向任务（理解 → 生成 QA）

Summary
论文整体摘要理解
Innovation
创新点提炼与解释
ExperimentSummary
实验结论归纳
ExperimentSteps
实验过程细节问答
Interpretation
结果解读与推断

🔄 逆向任务（受控退化 → 改写任务）

DraftPolish
给定草稿，生成精炼版本
LogicRefinement
逻辑跳跃版 → 严谨原文
CritiqueRevision
批评性重写，修复论证缺陷
HedgingTone
语气夸张版 → 学术措辞原文

🔁 每个 Agent 内部：ReAct 多轮推理循环

🤔 Think
分析任务&上下文
→
⚡ Act
调用LLM生成QA
→
🔍 Observe
检查输出格式/质量
→
✏️ Reflect
自我批评反思
→
🔄 Refine
修正重生成

⚖️ Quality Judge Agent（质量门控）

Q 有效性
问题是否清晰、有意义、不含答案
A 准确性
答案是否忠实于原文段落
逆向合规
退化版本是否真的有缺陷
语言质量
学术用语、逻辑连贯性
✅ 合格 → 写入数据库
❌ 不合格 → 反馈给 Agent 重生成（最多 N 轮）
🗄️ PostgreSQL · r5_d3_academic_writing 表 存储合格 QA pairs + 元信息（task_type / paper_id / score）
🎓 SFT 训练数据 导出为 r5_d3_academic_writing_sft.jsonl · 用于 Qwen 学术写作微调

"如何写好学术文章"
与随机噪声注入的区别：缺陷类型是语言学上有意义的（逻辑跳跃、语气夸张、结构混乱）， 而不是随机字符损坏。Quality Judge 会验证退化版本"真的更差"，保证训练信号的方向性。
